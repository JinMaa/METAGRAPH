traceblock

View Function

Gets execution traces for all transactions in a specified block. This method is particularly useful for block-level analysis of contract executions and understanding transaction patterns within a block.

Method Type:
Alkanes View Function
JSON-RPC Method:
metashrew_view
View Function:
traceblock
Required Parameters:
blockHeight, referenceBlock

Examples

Request
Response
cURL

Example Request

{ "method": "metashrew_view", "params": [ "traceblock", "0x00D70000", "latest" ], "id": 0, "jsonrpc": "2.0" }

Example Response

{ "jsonrpc": "2.0", "result": { "traces": { "a1b2c3d4e5f6...": { "success": true, "output": "0x...", "gas_used": 1234, "trace": [ { "pc": 0, "op": "OP_0", "stack": [""] }, // Additional trace steps ] }, // Additional transaction traces }, "block_height": "0x00D70000" }, "id": 0 }

cURL Command

curl -vvv https://mainnet.sandshrew.io/v2/lasereyes -X POST -d '{"method":"metashrew_view","params":["traceblock","0x00D70000","latest"],"id":0,"jsonrpc":"2.0"}' -H 'Content-Type: application/json'

Try It

Hex-encoded block height. Must include '0x' prefix and be properly padded (e.g., 0x00D70000)
Block to use as reference for state. Usually 'latest' or a newer block than the traced block.

Results will appear here after execution.

Notes

  • Block Height Format: Block height must be provided as a hex-encoded string with "0x" prefix (e.g., "0x00D70000").
  • Reference Block: The reference block parameter specifies which block's state to use when executing the traces. Usually this is set to "latest".
  • Response Size: The response can be quite large for blocks with many transactions. Consider implementing pagination or filtering if displaying in a UI.
  • Transaction Traces: The response includes traces for each transaction in the block, indexed by transaction ID.
  • Performance: This is a computationally intensive call. For production applications, consider caching results or using more targeted calls when possible.
  • Use with sandshrew_multicall: To efficiently fetch traces for multiple blocks, consider using the sandshrew_multicall method.
