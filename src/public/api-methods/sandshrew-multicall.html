sandshrew_multicall

JSON-RPC

Allows batching multiple JSON-RPC calls in a single request, reducing round trips and improving performance for complex data fetching operations.

Method Type:
JSON-RPC Method
JSON-RPC Method:
sandshrew_multicall

Example Request

{ "method": "sandshrew_multicall", "params": [ [ ["metashrew_height", []], ["btc_getblockcount", []], ["metashrew_view", ["protorunesbyheight", "0x0D6B38", "latest"]] ] ], "id": 0, "jsonrpc": "2.0" }

Notes

  • Parameter Format: The sandshrew_multicall method takes a single parameter: an array of calls, where each call is an array containing the method name and an array of parameters.
  • Response Structure: The response contains an array of results, with each result corresponding to the call at the same position in the request.
  • Error Handling: If any individual call fails, its corresponding result will contain an error object, but other calls will still be processed.
  • Efficiency: Use multicall when you need to make multiple related API calls, such as when calculating sync status (metashrew_height + btc_getblockcount) or fetching data across multiple blocks.
  • Request Size Limits: There may be limits on the number of calls you can batch in a single request. Start with smaller batches and increase as needed.
